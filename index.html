Framework Analyze mobile apps regarding Trust & Security issues

1. Availability of privacy policy

  • Is a privacy policy available somewhere?
  • Only available online? Any option to download? Any option to print?
  • Old versions of privacy policy (if policy is updated)

2. Accessibility of privacy policy

  • Is a privacy policy accessible on App Marketplace? Is it easy to find?
  • Is a privacy policy accessible within app? Is it easy to find?
  • Is the content of privacy policy well-designed?
  • Typography: readability
  • Structure: alignment/grouping of topics?

3. Privacy policy provides detailed information on collection of users’ data

Clearly stated

  • what kind of data is collected
  • why such data is collected (purpose)
  • if collection of such data is crucial

4. Privacy policy provides detailed information on use of users’ data

Clearly stated

  • for which purposes the data used
  • if use of such data is crucial

5. Privacy policy provides information on security of transactions

Provided if data exchanged between app and systems (servers/web services) is

  • is secured by any encryption mechanisms
  • is accessible by unauthorized persons/systems (Confidentiality)
  • might be changed (edit/delete) by unauthorized persons/systems (Integrity)

Provided if transaction process
  • is authenticated: confirmation of identity of participating parties (Authentication)
  • Credentials shouldn’t be stored on the phone file system
  • is authorized: validation of the authority of the transaction performed by providers (Authorization)
  • is guaranteed not to be subsequently repudiated (rejected) by e.g. Digital Signatures. This is not only to authenticate the parties, but also 'time stamp' the transaction. (Non-repudiation)

6. Privacy policy provides detailed information on sharing of users’ data

Clearly stated

  • whether data is shared with 3rd-parties
  • why such data is shared
  • if sharing of such data is crucial
  • how transfer of such data to 3rd-parties is secured (encryption)

7. Privacy policy provides information on storing of users’ data

Clearly stated

  • where is the data stored (on device/servers)
  • why is the data stored
  • if users can access to stored data about them
  • how such data is secured for storage (encryption)
  • how long is data stored
  • if stored old data (no longer in use) will be deleted

8. Privacy policy provides information on accountability

Is anyone responsible for privacy who should

  • review privacy policy before each app release
  • notify users if privacy policy is updated
  • confirm that users’ data is only available to which members who need to see it
  • answer privacy and security questions
  • Is contact information of responsible person available?
  • Email address and other communication channels
  • Work times (on which days and between which hours available)

9. Privacy policy provides information on permissions which needed by the app

  • Is there a general description of needed permissions?
  • Which permission does the app need and why?
  • Is there any option to choose between permissions within app?

10. Availability of options provided to users to choose for collection/use/sharing

Are there any options that users can decide and configure what kind of data might be

  • collected (for which purposes)
  • used (for which purposes)
  • shared (with which parties)
  • stored (for which purposes and how long)

11. Information about app on App Marketplace

User reviews and ratings

  • Issues stated by users in reviews
  • Average of rating in respect to other similar apps

Price
  • Is it for free?
  • Is it stated in description of app if all functionalities of app might be used for free without updating (purchasing) to another version?
  • Is there a another version of app (probably not free) that offers more functionalities?

Ads
  • Is it stated in description of app if any ads appear within app?

Permissions
  • Is there a general description of permissions?
  • Which permission does the app need and why?
  • Is there any option to choose between permissions during installation of app?
